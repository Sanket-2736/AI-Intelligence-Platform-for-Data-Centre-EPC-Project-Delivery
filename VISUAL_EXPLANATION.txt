PROBLEM: Upload Spinner Hangs Forever
======================================

User Flow (Broken):
┌─────────────────┐
│ Click Upload    │
└────────┬────────┘
         │
         ↓
    ┌─────────────────────────────────────────────────┐
    │ Frontend: setIngestingFiles(true)               │
    │ Spinner starts spinning...                      │
    └────────┬────────────────────────────────────────┘
             │
             ↓
    ┌─────────────────────────────────────────────────┐
    │ axios.post('/api/rfi/ingest/batch', formData)   │
    │ Waiting for response...                         │
    └────────┬────────────────────────────────────────┘
             │
             ↓
    🔴 🔴 🔴 HANG 🔴 🔴 🔴
    
    Backend is BLOCKED
    Response NEVER comes
    Promise NEVER resolves
    Spinner keeps SPINNING
    
    [After 30-60 seconds or never]
    ├─ Option A: Timeout error appears
    └─ Option B: Spinner just keeps spinning

WHY IT HANGS
============

Backend Architecture (Broken):

┌─────────────────────────────────────────────────────┐
│  FastAPI Async Endpoint (Main Event Loop Thread)   │
│  @router.post("/ingest/batch") async               │
│                                                     │
│  1. Save files ✓ (fast)                            │
│  2. Call ingest_multiple_documents()               │
│     └─ THIS IS SYNC, NOT AWAITED!                  │
│       └─ Call ingest_project_document()            │
│          └─ THIS IS SYNC, NOT AWAITED!             │
│            └─ Extract PDF ✓ (fast)                 │
│            └─ Chunk PDF ✓ (fast)                   │
│            └─ Call chroma.ingest_chunks()          │
│               └─ THIS IS SYNC, NOT AWAITED!        │
│                 └─ collection.upsert()             │
│                    └─ SentenceTransformer.encode() │
│                       └─ 🔴 BLOCKS HERE FOR 5-15min
│                          EVENT LOOP FROZEN        │
│                          CAN'T PROCESS REQUESTS   │
│                          CAN'T SEND RESPONSE      │
│                          SPINNER HANGS            │
│                                                     │
└─────────────────────────────────────────────────────┘

The Problem:
    ChromaDB.upsert()
    = Synchronous operation
    = Blocks calling thread
    = In async context
    = Blocks ENTIRE event loop
    = Nothing else can happen
    = Response never sent


THE FIX: asyncio.to_thread()
=============================

Backend Architecture (Fixed):

┌─────────────────────────────────────────────────────┐
│  FastAPI Async Endpoint (Main Event Loop Thread)   │
│  @router.post("/ingest/batch") async               │
│                                                     │
│  1. Save files ✓ (fast)                            │
│  2. await ingest_multiple_documents_async()        │ ← ASYNC!
│     └─ await ingest_project_document_async()       │ ← ASYNC!
│        └─ Extract PDF ✓ (fast)                     │
│        └─ Chunk PDF ✓ (fast)                       │
│        └─ await asyncio.to_thread(               │
│             lambda: chroma.ingest_chunks()         │
│           )                                        │
│           ├─ 🟢 SPAWNS BACKGROUND THREAD            │
│           │                                        │
│           └─ EVENT LOOP CONTINUES ✓               │
│              (Not blocked, responsive)             │
│              (Can send response NOW)               │
│              (Spinner stops NOW) ✓                 │
│                                                     │
│           Meanwhile in background thread:          │
│           └─ collection.upsert()                   │
│              └─ SentenceTransformer.encode()       │
│                 └─ Takes 5-15 seconds              │
│                    (In thread, not blocking)       │
│                                                     │
└─────────────────────────────────────────────────────┘

The Solution:
    asyncio.to_thread()
    = Offload blocking code
    = To background thread
    = Event loop stays responsive
    = Can send HTTP response
    = Meanwhile work happens in background
    = Frontend gets feedback immediately

===================

                    MAIN EVENT LOOP
                    ══════════════════════════
                    │                        │
                    │ Can handle:            │
                    │ • New HTTP requests    │
                    │ • Send responses       │

                    │                        │

                    └────────────────────────┘

                            │

                    │                  │

            │ Handle:      │   │ Handle:       │
            │ Embedding    │   │ (Other work)  │

            │              │   │               │
            └──────────────┘   └───────────────┘

Result: Blocking work doesn't stop main loop!


CODE DIFF - BEFORE vs AFTER
============================

BEFORE (Broken):
───────────────
@router.post("/ingest/batch")
async def ingest_batch_documents(...):
    result = ingest_multiple_documents(file_list)
                    ↑
                    └─ BLOCKING CALL - HANGS HERE


AFTER (Fixed):
──────────────
@router.post("/ingest/batch")
async def ingest_batch_documents(...):
    result = await ingest_multiple_documents_async(file_list)
                 ↑
                 └─ ASYNC CALL - RESPONSIVE


Inside async function:

BEFORE (Broken):
───────────────
ingest_result = chroma.ingest_chunks(...)
                ↑
                └─ BLOCKS EVENT LOOP


AFTER (Fixed):
──────────────
def _blocking_ingest():
    return chroma.ingest_chunks(...)

ingest_result = await asyncio.to_thread(_blocking_ingest)
                ↑
                └─ RUNS IN THREAD - DOESN'T BLOCK

KEY INSIGHT
===========

The embedding operation itself doesn't change.
It still takes 5-15 seconds.

What changes:
• WHERE it runs (background thread vs main thread)
• NOT blocking the HTTP response mechanism
• Frontend gets feedback immediately
• User sees progress, not frozen UI
